## Cursor vs. Claude — A Head-to-Head

🕒 *Estimated Time: 25 minutes*

---

## ✅ Your Task

Run the **same task** through both tools and feel the difference firsthand. By the end you'll have an instinct for which to reach for.

### 🧱 Stage 1: Pick one small project

Choose something concrete and self-contained, e.g.:

- A script that reads a CSV and makes one chart, **or**
- A small function plus a couple of tests, **or**
- A short data-cleaning routine.

### 🧱 Stage 2: Build it in Cursor

- [ ] Open the folder in Cursor.
- [ ] Use the chat (`Ctrl+L`) and inline edit (`Ctrl+I`) to scaffold and refine the code across files.
- [ ] Note: how does it handle **multiple files**? In-place edits? Autocomplete?

### 🧱 Stage 3: Work it through with Claude Desktop

- [ ] Describe the same goal to Claude Desktop and have a **conversation** about the design first.
- [ ] Paste your Cursor code and ask: *"What would a reviewer criticize? What's the cleaner structure?"*
- [ ] Note: how does it handle **reasoning and explanation** vs. direct file edits?

### 🧱 Stage 4: Reflect (3–5 sentences)

- [ ] Which tool felt better for **writing/refactoring code in place**?
- [ ] Which felt better for **thinking, explaining, and critiquing**?
- [ ] When would you combine them? (Hint: design in Claude → build in Cursor → review in Claude.)

---

## 💡 The takeaway

Most experienced users don't pick one — they **switch**. Cursor for hands-on editing across a codebase; Claude for reasoning, writing, and review. See [READ: Which Tool, When?](READ_cursor_vs_claude.md) for the full comparison.
