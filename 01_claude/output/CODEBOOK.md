# Codebook — `social_infra.csv`

One entry per column, generated by fanning out the `column-documenter` sub-agent (one instance per column) over [`../examples/social_infra.csv`](../examples/social_infra.csv), then assembled.

### `site_id`
- **Description:** unique identifier for each site.
- **Type:** text (id)
- **Units / levels:** n/a
- **Range or examples:** `S001`, `S002`, … `S008`
- **Missing convention:** none seen
- **Caveats:** none obvious.

### `name`
- **Description:** human-readable site name.
- **Type:** text
- **Units / levels:** n/a
- **Range or examples:** "Central Library", "Riverside Park"
- **Missing convention:** none seen
- **Caveats:** none obvious.

### `type`
- **Description:** category of social-infrastructure site.
- **Type:** categorical
- **Units / levels:** library, park, community_center, clinic
- **Range or examples:** 4 distinct levels
- **Missing convention:** none seen
- **Caveats:** none obvious.

### `lat` / `lon`
- **Description:** geographic coordinates (WGS84).
- **Type:** numeric (geographic)
- **Range or examples:** lat ≈ 42.42–42.47, lon ≈ −76.46 to −76.52
- **Missing convention:** none seen
- **Caveats:** all points cluster near Ithaca, NY — confirm that's expected.

### `capacity`
- **Description:** rough capacity of the site.
- **Type:** integer
- **Range or examples:** 0–400
- **Missing convention:** `0` appears for both parks (S002, S006)
- **Caveats:** ⚠️ those `0`s likely mean "not applicable," not a true zero — don't average them in blindly.

### `year_built`
- **Description:** year the site opened.
- **Type:** integer (year)
- **Range or examples:** 1955–2010
- **Missing convention:** blank for S003
- **Caveats:** one missing value (Eastside Community Center).

### `access_score`
- **Description:** accessibility score, 0–100.
- **Type:** integer
- **Range or examples:** 58–90 (excluding the sentinel)
- **Missing convention:** none blank
- **Caveats:** ⚠️ S004 = **999** is a sentinel/error, not a real score — exclude it before any analysis.

### `last_survey`
- **Description:** date the site was last surveyed.
- **Type:** date (YYYY-MM-DD)
- **Range or examples:** 2024-11-30 … 2025-03-05
- **Missing convention:** blank for S003 and S007
- **Caveats:** two sites never surveyed; treat their other fields with extra caution.
