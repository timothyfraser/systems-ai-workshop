# 📌 ACTIVITY

## Pi — A Local Coding Agent in the Terminal

🕒 *Estimated Time: 20 minutes*

---

## ✅ Your Task

Install **Pi**, a minimal, hackable coding-agent harness by **Mario Zechner** ([pi.dev](https://pi.dev/)), and point it at **Ollama** so a local model can read, write, edit, and run code on your machine — fully offline. Pi's whole core is **four tools** (read, write, edit, bash); everything else is a **Skill** or extension you add. That makes it the clearest way to *see* how a coding agent actually works.

> Pi completes the workshop's mental model: **Claude has Skills, Cursor has Skills/rules, Pi has Skills/extensions.** One idea, three tools.

### 🧱 Stage 1: Prerequisites

- [ ] Ollama serving — `curl http://localhost:11434/api/version`
- [ ] A coding model pulled — `ollama pull qwen2.5-coder:7b` (or `gemma4:e4b` on a light laptop)
- [ ] **Node.js** installed (Pi is an npm package) — get it from 🔗 [nodejs.org](https://nodejs.org/) if `node --version` fails

### 🧱 Stage 2: Install Pi

Pi installs from npm. Check the latest instructions at 🔗 [pi.dev](https://pi.dev/), then:

```bash
npm install -g @earendil-works/pi-coding-agent
pi --version
```

- [ ] `pi --version` prints a version.

### 🧱 Stage 3: Point Pi at Ollama

Pi reads provider config from `~/.pi/agent/models.json`. Add an **ollama** provider that uses Ollama's OpenAI-compatible endpoint:

```json
{
  "providers": {
    "ollama": {
      "baseUrl": "http://localhost:11434/v1",
      "api": "openai-completions",
      "apiKey": "ollama",
      "models": [
        { "id": "qwen2.5-coder:7b" },
        { "id": "gemma4:e4b" }
      ]
    }
  }
}
```

- [ ] `baseUrl` is Ollama's local endpoint with `/v1` on the end.
- [ ] `apiKey` is the literal string `"ollama"` — Ollama ignores it, but the field must be present.
- [ ] If Pi complains about an unsupported `developer` role, add `"compat": { "supportsDeveloperRole": false }` inside the provider.

### 🧱 Stage 4: Run it

From inside a project folder:

```bash
pi
```

- [ ] Use the **`/model`** command to pick one of your Ollama models (Pi reloads the config each time — no restart needed).
- [ ] Ask for something small and concrete:
  ```
  Read utils.py, add type hints and a docstring to each function, and show me the diff.
  ```
- [ ] Pi proposes edits; review and approve. **Read the diff before accepting** — see [zero-trust](READ_security.md).

### 🧱 Stage 5: Prove it's local

- [ ] Turn off Wi-Fi and run another small task. It still works. 🔌🚫

## ⚠️ Local-model reality check

Small local models aren't frontier models. Set yourself up to win:

- **Use the biggest coder that fits** (`qwen2.5-coder:14b`, or `qwen3-coder:30b` on 32 GB).
- **One file, one clear instruction** at a time — local models lose the thread on sprawling edits.
- **Commit first.** A clean git tree is your undo button if an edit goes wrong.

## 🔁 Prefer a different harness?

Pi, **OpenCode**, and **Aider** all point at the same `localhost:11434` endpoint — what you learn here transfers. See [the harness landscape](READ_harnesses.md) to compare. Want a harness *inside your editor* instead of the terminal? Use [Continue](ACTIVITY_continue_local.md).
