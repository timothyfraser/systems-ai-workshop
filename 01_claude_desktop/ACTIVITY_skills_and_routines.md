# 📌 ACTIVITY

## Skills & Routines for Repetitive Work

🕒 *Estimated Time: 20 minutes*

---

## ✅ Your Task

Capture a repetitive academic task as a reusable **routine** so you never have to re-explain it. Think: drafting reference-letter outlines, formatting weekly announcements, turning meeting notes into action items, or summarizing student feedback.

### 🧱 Stage 1: Pick a recurring chore

- [ ] Choose something you do often with a predictable shape. Good candidates:
  - Turning bullet notes into a polished email
  - Summarizing a batch of student responses into themes
  - Drafting a structured outline for a recommendation letter
  - Converting a meeting transcript into decisions + owners + dates

### 🧱 Stage 2: Write it down as a Skill

A **Skill** is a small folder with a `SKILL.md` file: a name, a description of *when* to use it, and step-by-step instructions. See this repo's [`claude-desktop-routines`](../.claude/skills/claude-desktop-routines/SKILL.md) Skill — it walks Claude through designing a good routine, and you can copy its shape.

- [ ] Ask Claude: *"Help me turn this task into a reusable Skill. Here's what I do and what good output looks like…"*
- [ ] Save the result as `.claude/skills/<your-skill>/SKILL.md`.

### 🧱 Stage 3: Use it and refine

- [ ] Run the routine on a fresh real example.
- [ ] Note where the output missed; fold those fixes back into the `SKILL.md`.
- [ ] A good routine gets sharper every time you use it.

---

## 💡 Skills vs. one-off prompts

A clever prompt helps once. A **Skill** is that prompt, saved, named, and improved over time — so the next person (or the next-month you) gets the same quality without rediscovering it.

Build one routine for a task you'll repeat this semester, then use it twice and tighten the instructions.
